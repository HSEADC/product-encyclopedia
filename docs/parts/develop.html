Разработка
Третий этап — разработка, является первым шагом в пространстве решений. На этом этапе создаются прототипы решения, тестируются с пользователем и разрабатывается дальнейший план работы над продуктом.

Разработка — это первый шаг в пространстве решений. На этом этапе команда создает прототипы решения и тестирует его, вовлекая конечного пользователя в процесс тестирования с помощью дивергентного подхода, что означает, что все типы решений являются прототипами.
Стадия разработки включает в себя большую междисциплинарную работу — объединение дизайнеров с внутренними партнерами, такими как инженеры, разработчики или другие отделы, обладающие опытом, необходимым для проекта. Большой выигрыш от этого аспекта стадии разработки заключается в том, что, объединяя различные отделы, ускоряется процесс решения проблем.
Все методы, используемые на данном этапе, нацелены на один и тот же результат: эффективно использовать инструмент прототипирования и воплотить решение в жизнь.
На этапе разработки главная цель — сгенерировать потенциальные решения для определенной проблемы. Точно так же, как были расширены исследования, необходимо расширить возможности решений. Разработка — это этап, на котором будут генерироваться идеи, основанные на первом «алмазе». Стоит искать вдохновение из разных источников и делиться процессом с как можно большим количеством заинтересованных сторон.

Для правильного проведения такого рода исследований требуется время и внимание к пользовательскому опыту каждого отдельного пользователя, чтобы понять, что вызывает поведение, наблюдаемое во время тестирования юзабилити. Такого рода исследования раскрывают больше данных, когда есть возможность задать потенциальным пользователям вопросы и не упускать возможности собрать больше данных из каждого исследования, которые могут быть использованы для оценки продукта.

Roadmap
Дорожная карта — это визуальное представление стратегических планов. Она связывает воедино стратегию («почему»), работу, которую нужно будет выполнить для достижения целей («что»), и график выполнения («когда»).

Lean canvas
Методология бережливого производства (lean methodology), известная внедрением различных инструментов управления продуктами, таких как lean canvas, стала популярной после того, как упомянутые гиганты уже были на пути к успеху. И, скорее всего, их истории легли в основу этого продвинутого мышления.
Разработка
User Flow
User Flow — это карта, показывающая путь, который пользова- тель выберет в приложении для выполнения задачи. UserFlow создается для интуитивно понятного проектирования продуктов, предоставляя пользователям правильную информацию в нужное время и позволяя им выполнять желаемые задачи за минимальное количество шагов, насколько это возможно.

User Stories
Продуктовые команды предпочитают разбивать работу по разработке на пользовательские истории, а не на характеристики продукта или требования к продукту по нескольким причинам.
